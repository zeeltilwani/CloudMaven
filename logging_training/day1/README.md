# 📘 Day 1 – Logging Fundamentals

## 🚀 What I Learned

On Day 1, I understood the basics of **Logging** and why it is a very important concept in real-world systems.

### 🔍 What is Logging?
Logging means **recording events that happen inside a system or application**.  
It helps us understand what is going on inside the system at any point in time.

📌 Example:
- ATM transactions
- Bank SMS alerts
- Website activity logs

---

## 💡 Why Logging is Important

Logging plays a key role in:

- 🐞 Debugging issues when something breaks
- 👤 Tracking user activity
- 🔐 Security auditing
- ⚡ Performance analysis

Without logging, it becomes very difficult to understand system behavior.

---

## ⚖️ Monitoring vs Logging

| Monitoring | Logging |
|-----------|--------|
| Works on metrics | Works on events |
| Real-time alerts | Historical records |
| CPU, Memory usage | Errors, Requests |

👉 Monitoring tells *something is wrong*  
👉 Logging tells *what exactly went wrong*

---

## 🧾 Types of Logs

I learned about different types of logs used in systems:

- Application Logs
- System Logs
- Access Logs
- Error Logs
- Security Logs

---

## 📊 Log Levels

Logs are categorized based on severity:

- DEBUG → Detailed info for developers  
- INFO → General information  
- WARN → Warning messages  
- ERROR → Something failed  
- FATAL → Critical failure  

📌 Example:2026-04-06 10:00:00 INFO User logged in
2026-04-06 10:01:00 ERROR Database connection failed


---

## 🔄 Log Flow Architecture

I understood how logs move in a system:


Application → Log File → Log Agent → Storage → Dashboard


This helps in collecting and visualizing logs centrally.

---

## ⚠️ Problems Without Logging

If logging is not present:

- No visibility into system
- Hard to debug issues
- Increased downtime
- Blind troubleshooting

---

## 🧠 Final Understanding

Logging is a **critical part of any system**.  
It helps in debugging, monitoring, and maintaining system reliability.
